자바스크립트

웹 브라우저상 가장 많이 기본적으로 사용하는 스크립트 언어
컴파일 방식 => 컴파일 단계를 거치고 나서 실행되는 개념
인터프리터 방식=> 컴파일 단계를 거치지 않고 곧바로 실행되는 개념 자바/자바스크립느 == 컴파일 + 인터프리터 방식(JIT Compiler)

자바와의 차이점

  • 함수를 일급객체로 취급함 : 함수를 호출할때 값으로 호출함
  • 문법 자체가 단순한 구조, 원칙을 가지고 있기 때문에 쉬움 예)접근제한자 개념 없음,함수 정의 시 반환형 정의 없음, 변수 선언 시 자료형을 지정하지 않음(타입추론)
  • 확장성이 좋아 배워두면 여기저기 쓸 수 있음

자바스크립트 소스코드 작성 위치에 따른 종류

  1. inline(인라인) : 요소 속성에 간단한 코드를 작성해서 실행
  2. internal(내부) : HTML문서내에 script태그에 코드를 작성헤서 실행
  3. external(외부) : 코드를 별도의 .js파일로 작성해서 읽어오는 방법
    (모듈파일에서 export 사용하려는 쪽에서 import해서 쓸 수도 있음)

inline방식

        태그에 이벤트 속성에 실행할 간단한 소스코드를 작성하는 방식 

        코드가 매우 소량이거나  또는 다른 방식과 혼합해서 사용해야 할 때 소려해 볼 수 있음

        [표현법]
        <태그명 이벤트 속성="실행하려는 소스코드">
        이벤트 속성 앞은 항상 on이 붙는다. 그 뒤에 속에 들어갈 값을 넣어줌

2) internal 방식

        HTML문서에 script라는 태그 영역을 만든 뒤
        함수단위로 소스코드를 작성하고 
        특정 요소의 이벤트  발생 시 해당 함수를 호출하는 방식

        => script 태그 자체는 head , body 문서내부 어디에다가 적어도 잘 동작하지만, 브라우저 성능 향상을 위해 항상 스크립트 코드는
        외부방식을 사용하여 html요소 가장 하단에 연결 시키는것이 좋음.
